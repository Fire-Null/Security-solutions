
#### اهمیت مدیریت خطا از آنجایی نشئت می­گیرد که خطاهای تولیدشده مانند اطلاعات انگشت‌نگاری می­توانند اطلاعاتی در خصوص تکنولوژی­های استفاده‌شده در پیاده­سازی برنامه کاربردی افشا نمایند. این اطلاعات به فرد مهاجم کمک کند تا شناخت دقیق‌تری از سامانه کسب کند و حملات خود را با دقت و هدفمندی بیشتری انجام دهد. ازاین‌رو هر سامانه می­بایست هرگونه خطا اعم از خطاهای سیستمی، خطاهای I/O و غیره را مدیریت نماید.
* مدیریت خطا درواقع به معنیِ عدم نمایش جزئیات خطا به کاربر بوده. به‌طور عمومی پیشنهاد می­شود یک صفحه مخصوص اعلام بروز خطا ساخته شود و در صورت بروز هرگونه خطا کاربر به این صفحه هدایت شود و از نشان­ دادن جزئیات خطا به کاربر خود­داری شود.
* در این بخش مانند قبل به تفکیک تکنولوژی­های استفاده شده در پیاده‌سازی برنامه کاربردی، روش­های مدیریت خطا در سطح ارائه خواهند شد.
>به‌منظور مدیریت خطاهای HTTP می‌توان از تنظیمات زیر استفاده کرد. نمونه زیر برای مدیریت خطای HTTP 404 نشان داده شده است که می‌توان آن را برای خطاهای HTTP دیگر بسط داد.
```xml
<web-app xmlns:xsi=http://www.w3.org/2001/XMLSchema-instance
Xmlns=http://java.sun.com/xml/ns/javaee
Xsi: schemaLocation=”http://java.sun.com/xml/ns/javaee
  http:java.sun.com/xml/ns/javaee/web-app_3_1.xsd"
version = “3.1” >

<error-page>
    <error-code>404<error-code>
     <location>/error.jsp</location> <!—
/src/main/webapp/error-404.html-->
 </error-page>
</web-app>
```

>منظور از خطاهای سرور درواقع خطاهای HTTP می­باشد. با وجود اینکه این نوع خطاها را در سطح برنامه­ی کاربردی نیز می­توان مدیریت نمود اما به منظور یک پارچگی مطالب در این بخش راهکارهای مدیریت خطا بر اساس نوع وب سرور استفاده شده دسته بندی شده و راه‌حل‌های ارائه شده بر اساس تنظیمات سرور می­باشد.
